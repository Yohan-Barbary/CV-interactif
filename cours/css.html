CSS3 - Les Fondamentaux

Introduction au CSS

CSS (Cascading Style Sheets) est un langage de style utilisé pour décrire la présentation d'un document HTML.

Méthodes d'intégration CSS :

  • Inline CSS (dans l'attribut style)
  • Internal CSS (dans la balise style)
  • External CSS (fichier .css externe)

Syntaxe de base


/* Sélecteur externe (dans un fichier .css) */
.ma-classe {
    color: blue;
    font-size: 16px;
    margin: 10px;
}

#mon-id {
    background-color: #f0f0f0;
}

div {
    padding: 20px;
}

Spécificité CSS


/* Spécificité croissante */
h1 { /* Spécificité: 0,0,1 */
    color: blue;
}

.titre { /* Spécificité: 0,1,0 */
    color: red;
}

#header h1 { /* Spécificité: 1,0,1 */
    color: green;
}

/* Héritage */
body {
    font-family: Arial, sans-serif;
    color: #333;
}

/* Valeurs initiales et héritées */
.container {
    color: inherit;
    border: initial;
}

/* Unités CSS courantes */
.element {
    /* Unités absolues */
    width: 100px;
    margin: 1cm;
    
    /* Unités relatives */
    font-size: 1.2em;
    padding: 2rem;
    width: 50%;
    
    /* Unités viewport */
    height: 100vh;
    width: 50vw;
}

Concepts avancés :

  • La spécificité détermine quel style sera appliqué en cas de conflit
  • L'héritage permet la transmission des propriétés aux éléments enfants
  • Les unités relatives s'adaptent au contexte

Calcul de la spécificité :

  • Style inline : 1,0,0,0
  • ID : 0,1,0,0
  • Classes, attributs, pseudo-classes : 0,0,1,0
  • Éléments, pseudo-éléments : 0,0,0,1

Bonnes pratiques d'organisation :

  • Utiliser une méthodologie de nommage (BEM, SMACSS, OOCSS)
  • Structurer les fichiers CSS par composants
  • Minimiser l'utilisation des !important
  • Commenter le code pour la maintenance

Sélecteurs CSS

Types de sélecteurs :

  • Sélecteurs simples (élément, classe, id)
  • Sélecteurs combinateurs
  • Sélecteurs de pseudo-classes
  • Sélecteurs d'attributs

Exemples de sélecteurs


/* Sélecteur d'élément */
p {
    color: blue;
}

/* Sélecteur de classe */
.ma-classe {
    background-color: gray;
}

/* Sélecteur d'ID */
#mon-id {
    border: 1px solid black;
}

/* Sélecteur descendant */
div p {
    margin: 10px;
}

/* Sélecteur d'enfant direct */
div > p {
    padding: 5px;
}

/* Pseudo-classes */
a:hover {
    color: red;
}

/* Sélecteur d'attribut */
input[type="text"] {
    border-radius: 5px;
}

Sélecteurs avancés et combinaisons


/* Sélecteurs d'attributs avancés */
[class*="btn-"] { /* Contient "btn-" */
    padding: 10px 20px;
}

[lang|="en"] { /* Commence par "en" suivi de "-" */
    font-style: italic;
}

/* Pseudo-classes structurelles */
li:nth-child(odd) {
    background: #f5f5f5;
}

p:first-of-type {
    font-size: 1.2em;
}

/* Combinaisons complexes */
.sidebar h2:not(.exclude) {
    border-bottom: 2px solid currentColor;
}

article > *:not(h2):not(h3) {
    margin-left: 1rem;
}

/* Sélecteurs de performance */
.menu .active { /* Plus performant que .menu li.active */
    font-weight: bold;
}

/* Nouveaux sélecteurs modernes */
:is(h1, h2, h3):hover {
    color: var(--accent-color);
}

.container:has(> img) {
    display: grid;
}

Utilisations avancées :

  • Les sélecteurs d'attributs pour cibler des éléments spécifiques
  • Les pseudo-classes pour des sélections basées sur l'état ou la position
  • Les combinateurs pour des relations parent-enfant précises

Optimisation des sélecteurs :

  • Éviter les sélecteurs trop profonds (max 3 niveaux)
  • Privilégier les classes aux sélecteurs d'éléments
  • Utiliser les nouveaux sélecteurs pour simplifier le code

Cas d'utilisation courants :

  • Stylisation conditionnelle basée sur les attributs
  • Mise en forme alternée avec nth-child
  • Exclusion d'éléments spécifiques avec :not()
  • Détection de contenu avec :has()

Box Model

Composants du Box Model :

  • Content (contenu)
  • Padding (rembourrage)
  • Border (bordure)
  • Margin (marge)

.element {
    /* Content */
    width: 300px;
    height: 200px;

    /* Padding */
    padding: 20px;
    /* ou */
    padding-top: 10px;
    padding-right: 20px;
    padding-bottom: 10px;
    padding-left: 20px;

    /* Border */
    border: 1px solid black;
    /* ou */
    border-width: 1px;
    border-style: solid;
    border-color: black;

    /* Margin */
    margin: 10px;
    /* ou */
    margin: 10px 20px; /* vertical horizontal */
    margin: 10px 20px 15px 25px; /* top right bottom left */
}

/* Box-sizing */
* {
    box-sizing: border-box; /* inclut padding et border dans width/height */
}

Concept clé :

Le Box Model est fondamental pour comprendre la mise en page CSS :

  • Le contenu est au centre de la boîte
  • Le padding crée de l'espace autour du contenu
  • La bordure encadre le padding
  • La marge crée de l'espace entre les éléments

Conseil pratique :

  • Utilisez box-sizing: border-box pour un calcul plus intuitif des dimensions
  • Les marges verticales peuvent se chevaucher (margin collapse)
  • Le padding affecte la taille finale de l'élément

Flexbox Layout

Concepts Flexbox :

  • Conteneur flex (flex container)
  • Éléments flex (flex items)
  • Axes principal et secondaire
  • Distribution de l'espace

/* Conteneur Flex de base */
.flex-container {
    display: flex;
    flex-direction: row; /* row | column | row-reverse | column-reverse */
    flex-wrap: wrap; /* nowrap | wrap | wrap-reverse */
    gap: 1rem;
}

/* Alignement sur l'axe principal */
.justify-content {
    justify-content: space-between; /* flex-start | center | space-around | space-evenly */
}

/* Alignement sur l'axe secondaire */
.align-items {
    align-items: center; /* flex-start | flex-end | stretch | baseline */
}

/* Propriétés des éléments flex */
.flex-item {
    flex: 1 1 300px; /* grow shrink basis */
    align-self: flex-start;
    order: 1;
}

/* Patterns Flexbox courants */
/* Navigation responsive */
.navbar {
    display: flex;
    justify-content: space-between;
    align-items: center;
    padding: 1rem;
}

/* Grille de cartes */
.card-grid {
    display: flex;
    flex-wrap: wrap;
    gap: 1rem;
}

.card {
    flex: 1 1 300px;
    max-width: calc(33.333% - 1rem);
}

/* Layout Holy Grail */
.holy-grail {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}

.holy-grail-body {
    display: flex;
    flex: 1;
}

Cas d'utilisation :

  • Barres de navigation
  • Mise en page de cartes
  • Centrage vertical et horizontal
  • Layouts responsives

Bonnes pratiques :

  • Utiliser flex-basis plutôt que width
  • Préférer gap à margin pour l'espacement
  • Combiner avec media queries pour la responsivité
  • Éviter les flex-direction: column sur mobile

CSS Grid Layout

Concepts Grid :

  • Lignes et colonnes
  • Zones de grille
  • Espacement (gap)
  • Alignement

/* Conteneur Grid de base */
.grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    grid-gap: 1rem;
}

/* Grid Areas */
.layout {
    display: grid;
    grid-template-areas: 
        "header header header"
        "nav    main   aside"
        "footer footer footer";
    grid-template-columns: 200px 1fr 200px;
    grid-template-rows: auto 1fr auto;
}

/* Positionnement des éléments */
.header { grid-area: header; }
.nav { grid-area: nav; }
.main { grid-area: main; }

/* Grid avancé */
.advanced-grid {
        display: grid;
    grid-template-columns: [start] 1fr [content-start] 900px [content-end] 1fr [end];
    grid-auto-rows: minmax(100px, auto);
}

/* Responsive Grid */
@media (max-width: 768px) {
    .layout {
        grid-template-areas:
            "header"
            "nav"
            "main"
            "aside"
            "footer";
        grid-template-columns: 1fr;
    }
}

/* Grid avec auto-placement */
.auto-grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
    grid-auto-flow: dense;
}

Avantages de Grid :

  • Contrôle bidimensionnel du layout
  • Placement précis des éléments
  • Responsive sans media queries
  • Gestion simplifiée des espaces

Patterns courants :

  • Layouts de page complets
  • Grilles de photos/cartes
  • Systèmes de colonnes
  • Layouts asymétriques

Design Responsive

Concepts clés :

  • Mobile First
  • Media Queries
  • Breakpoints
  • Unités fluides

/* Approche Mobile First */
.container {
    width: 100%;
    padding: 1rem;
    font-size: 16px;
}

/* Media Queries - Breakpoints standards */
@media (min-width: 576px) {
    .container {
        max-width: 540px;
    }
}

@media (min-width: 768px) {
    .container {
        max-width: 720px;
    }
}

/* Unités fluides avec clamp() */
.fluid-text {
    font-size: clamp(1rem, 2vw + 0.5rem, 2rem);
}

/* Layout responsive avec Grid */
.grid-responsive {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    gap: 1rem;
}

/* Images responsives */
img {
    max-width: 100%;
    height: auto;
}

/* Navigation responsive */
.nav-mobile {
    display: block;
}

.nav-desktop {
    display: none;
}

@media (min-width: 992px) {
    .nav-mobile {
        display: none;
    }
    
    .nav-desktop {
        display: flex;
    }
}

Bonnes pratiques :

  • Commencer par le mobile (Mobile First)
  • Utiliser des breakpoints standards
  • Préférer les unités relatives (rem, em, %)
  • Tester sur différents appareils

Variables CSS (Custom Properties)

Avantages :

  • Réutilisabilité
  • Maintenance facilitée
  • Thèmes dynamiques
  • Scope CSS

/* Déclaration des variables globales */
:root {
    /* Couleurs */
    --primary-color: #007bff;
    --secondary-color: #6c757d;
    --success-color: #28a745;
    
    /* Typographie */
    --font-main: 'Arial', sans-serif;
    --font-size-base: 16px;
    --line-height: 1.5;
    
    /* Espacements */
    --spacing-unit: 8px;
    --container-padding: calc(var(--spacing-unit) * 2);
}

/* Utilisation des variables */
.button {
    background-color: var(--primary-color);
    padding: var(--spacing-unit) calc(var(--spacing-unit) * 2);
    font-family: var(--font-main);
}

/* Thème sombre */
@media (prefers-color-scheme: dark) {
    :root {
        --primary-color: #66b3ff;
        --text-color: #ffffff;
        --bg-color: #1a1a1a;
    }
}

/* Variables locales */
.card {
    --card-padding: 1.5rem;
    padding: var(--card-padding);
}

/* Fallbacks */
.element {
    color: #007bff; /* Fallback */
    color: var(--primary-color, #007bff);
}

Cas d'utilisation :

  • Systèmes de thèmes
  • Composants réutilisables
  • Gestion des couleurs
  • Responsive design

Bonnes pratiques :

  • Nommer les variables de manière descriptive
  • Organiser les variables par catégories
  • Utiliser des fallbacks
  • Documenter les variables importantes

Animations et Transitions CSS

Types d'animations :

  • Transitions simples
  • Animations keyframes
  • Animations d'interface
  • Effets interactifs

Exemples d'animations courantes


/* Animation de chargement (Spinner) */
@keyframes spinner {
    0% {
        transform: rotate(0deg);
    }
    100% {
        transform: rotate(360deg);
    }
}

.loading-spinner {
    width: 40px;
    height: 40px;
    border: 4px solid #f3f3f3;
    border-top: 4px solid #3498db;
    border-radius: 50%;
    animation: spinner 1s linear infinite;
}

/* Animation de fade-in */
@keyframes fadeIn {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

.fade-in {
    animation: fadeIn 0.5s ease-out forwards;
}

/* Animation de menu hamburger */
.hamburger-line {
    transition: transform 0.3s ease-in-out;
}

.hamburger.active .line-1 {
    transform: translateY(8px) rotate(45deg);
}

.hamburger.active .line-2 {
    opacity: 0;
}

.hamburger.active .line-3 {
    transform: translateY(-8px) rotate(-45deg);
}

/* Animation de carte flip */
.card {
    perspective: 1000px;
    transform-style: preserve-3d;
    transition: transform 0.6s;
}

.card:hover {
    transform: rotateY(180deg);
}

/* Animation de skeleton loading */
@keyframes shimmer {
    0% {
        background-position: -200% 0;
    }
    100% {
        background-position: 200% 0;
    }
}

.skeleton {
    background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
    background-size: 200% 100%;
    animation: shimmer 1.5s infinite;
}

/* Animation de notification */
@keyframes notification {
    0% {
        transform: translateX(100%);
    }
    10% {
        transform: translateX(0);
    }
    90% {
        transform: translateX(0);
    }
    100% {
        transform: translateX(100%);
    }
}

.notification {
    animation: notification 4s ease-in-out;
}

Propriétés d'animation importantes :

  • animation-duration : durée de l'animation
  • animation-timing-function : courbe d'accélération
  • animation-delay : délai avant le début
  • animation-iteration-count : nombre de répétitions
  • animation-direction : direction de l'animation
  • animation-fill-mode : état avant/après l'animation

Bonnes pratiques :

  • Privilégier transform et opacity pour les performances
  • Utiliser des animations courtes (< 400ms) pour l'UI
  • Éviter les animations qui peuvent donner le mal des transports
  • Respecter prefers-reduced-motion

Cas d'utilisation courants :

  • Feedback utilisateur (hover, click, etc.)
  • États de chargement
  • Transitions de page
  • Micro-interactions
  • Animations de contenu

Transformations et Effets Avancés CSS

Types de transformations :

  • Transformations 2D
  • Transformations 3D
  • Perspectives
  • Effets composés

Transformations 2D de base


/* Transformations de base */
.translate {
    transform: translate(50px, 20px);
}

.scale {
    transform: scale(1.5);
}

.rotate {
    transform: rotate(45deg);
}

.skew {
    transform: skew(10deg, 20deg);
}

/* Transformations combinées */
.combined {
    transform: rotate(45deg) scale(1.5) translateX(50px);
}

/* Point d'origine de transformation */
.origin {
    transform-origin: left top;
    transform: rotate(45deg);
}

Transformations 3D avancées


/* Configuration 3D */
.scene {
    perspective: 1000px;
    perspective-origin: 50% 50%;
}

.container-3d {
    transform-style: preserve-3d;
}

/* Cube 3D */
.cube {
    position: relative;
    width: 200px;
    height: 200px;
    transform-style: preserve-3d;
    animation: rotate3d 5s infinite linear;
}

.cube-face {
    position: absolute;
    width: 100%;
    height: 100%;
    backface-visibility: hidden;
}

.front { transform: translateZ(100px); }
.back { transform: rotateY(180deg) translateZ(100px); }
.right { transform: rotateY(90deg) translateZ(100px); }
.left { transform: rotateY(-90deg) translateZ(100px); }
.top { transform: rotateX(90deg) translateZ(100px); }
.bottom { transform: rotateX(-90deg) translateZ(100px); }

/* Effets de carte avec flip */
.card-container {
    perspective: 1000px;
}

.card-3d {
    position: relative;
    transform-style: preserve-3d;
    transition: transform 0.8s;
}

.card-3d:hover {
    transform: rotateY(180deg);
}

/* Effets parallax */
.parallax-container {
    perspective: 1px;
    height: 100vh;
    overflow-x: hidden;
    overflow-y: scroll;
}

.parallax-layer {
    position: absolute;
    top: 0;
    right: 0;
    bottom: 0;
    left: 0;
}

.layer-back {
    transform: translateZ(-1px) scale(2);
}

.layer-base {
    transform: translateZ(0);
}

.layer-front {
    transform: translateZ(1px) scale(0.5);
}

Points clés :

  • Les transformations 3D nécessitent une perspective
  • transform-style: preserve-3d maintient la hiérarchie 3D
  • backface-visibility contrôle la visibilité du verso
  • Les transformations peuvent être animées

Performances :

  • Utiliser transform plutôt que position absolute
  • Activer l'accélération matérielle avec translateZ(0)
  • Éviter les transformations sur des éléments trop grands
  • Optimiser les animations avec will-change

Cas d'utilisation :

  • Cartes interactives
  • Menus et navigation 3D
  • Effets de parallaxe
  • Transitions de page
  • Galeries d'images

Techniques avancées :

  • CSS Grid Areas
  • Masonry Layout
  • Multi-column Layout
  • Sticky Positioning

/* Layout avec Grid Areas */
.grid-container {
    display: grid;

.no-break {
    break-inside: avoid;
    page-break-inside: avoid;

.sticky-container {

    overflow-y: auto;

.sticky-element {

.nested-grid {

    display: grid;

/* Container Queries */
@container (min-width: 400px) {
    .responsive-component {
        grid-template-columns: 1fr 1fr;
    }
}

Layouts adaptatifs avancés


/* Layout avec CSS Shapes */
.shape-outside {
    float: left;
    shape-outside: circle(50%);
    clip-path: circle(50%);
    width: 200px;
    height: 200px;
}

/* Aspect Ratio Boxes */
.aspect-ratio-box {
    aspect-ratio: 16 / 9;
    width: 100%;
}

/* CSS Grid Auto-fit vs Auto-fill */
.auto-fit {
    grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
}

.auto-fill {
    grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}

Techniques avancées :

  • Grid Areas pour des layouts complexes
  • Masonry pour des grilles dynamiques
  • Multi-column pour le contenu textuel
  • Sticky positioning pour les éléments fixes

Bonnes pratiques :

  • Utiliser des unités relatives pour la flexibilité
  • Combiner Grid et Flexbox selon les besoins
  • Prévoir les cas de débordement
  • Tester sur différentes tailles d'écran

Considérations de performance :

  • Éviter les calculs de layout trop fréquents
  • Utiliser contain: layout pour isoler les composants
  • Optimiser les animations de layout
  • Préférer les propriétés composites

Sélecteurs Avancés et Pseudo-éléments

Types de sélecteurs avancés :

  • Pseudo-éléments (::before, ::after)
  • Sélecteurs relationnels
  • Sélecteurs de négation et matches
  • Sélecteurs d'attributs complexes

Pseudo-éléments avancés


/* Création de contenu décoratif */
.quote::before {
    content: """;
    font-size: 3em;
    color: var(--accent-color);
    position: absolute;
    left: -0.5em;
}

/* Styles de première ligne et lettre */
.article p::first-letter {
    font-size: 2em;
    float: left;
    margin-right: 0.2em;
    line-height: 1;
}

.article p::first-line {
    font-weight: bold;
    color: var(--emphasis-color);
}

/* Sélection de texte personnalisée */
::selection {
    background: var(--brand-color);
    color: white;
}

/* Marqueurs de liste personnalisés */
.custom-list li::marker {
    content: "👉 ";
    color: var(--accent-color);
}

/* Effets de superposition avec ::before */
.overlay-effect::before {
    content: "";
    position: absolute;
    inset: 0;
    background: linear-gradient(to bottom, transparent, rgba(0,0,0,0.7));
    opacity: 0;
    transition: opacity 0.3s;
}

.overlay-effect:hover::before {
    opacity: 1;
}

Sélecteurs relationnels modernes


/* Sélecteur :has() */
article:has(> img) {
    padding: 0;
}

form:has(:invalid) {
    border-color: red;
}

/* Combinaison :is() et :where() */
:is(article, section) :where(h1, h2, h3) {
    color: var(--heading-color);
}

/* Sélecteurs de négation avancés */
.menu > :not(li, .separator) {
    margin: 1rem;
}

/* Sélecteurs d'attributs complexes */
[data-state*="active"] {
    background: var(--active-bg);
}

[href^="https"][href*="example.com"] {
    color: var(--external-link-color);
}

Utilisations courantes :

  • Création d'icônes et décorations
  • Effets de superposition
  • Styles typographiques avancés
  • Validation de formulaires
  • Mise en page conditionnelle

Bonnes pratiques :

  • Utiliser ::before/::after pour le contenu décoratif uniquement
  • Préférer les attributs data- pour les états
  • Éviter la surcharge de pseudo-éléments
  • Maintenir l'accessibilité du contenu

Considérations de performance :

  • Les pseudo-éléments créent des éléments supplémentaires dans le DOM
  • Limiter l'utilisation des sélecteurs complexes
  • Optimiser les sélecteurs pour le rendu
  • Utiliser des classes plutôt que des sélecteurs d'attributs quand possible

Compatibilité :

  • Vérifier le support des nouveaux sélecteurs (:has, :is)
  • Prévoir des fallbacks pour les navigateurs anciens
  • Tester sur différents navigateurs
  • Utiliser @supports pour les fonctionnalités avancées

Accessibilité et CSS

Points clés de l'accessibilité :

  • Contraste et lisibilité
  • Navigation au clavier
  • Préférences utilisateur
  • États interactifs

Contraste et Lisibilité


/* Variables pour les couleurs accessibles */
:root {
    --text-color: #333333; /* Ratio de contraste > 4.5:1 */
    --link-color: #0066cc; /* Ratio de contraste > 4.5:1 */
    --focus-outline: #2196f3;
}

/* Taille de texte adaptative */
body {
    font-size: clamp(1rem, 2vw, 1.2rem);
    line-height: 1.5;
    font-family: system-ui, -apple-system, sans-serif;
}

/* Focus visible */
:focus-visible {
    outline: 3px solid var(--focus-outline);
    outline-offset: 2px;
}

/* Suppression du focus pour les éléments non-interactifs */
[tabindex="-1"] {
    outline: none;
}

Préférences Utilisateur


/* Mode sombre */
@media (prefers-color-scheme: dark) {
    :root {
        --text-color: #f0f0f0;
        --background-color: #1a1a1a;
        --link-color: #66b3ff;
    }
}

/* Réduction de mouvement */
@media (prefers-reduced-motion: reduce) {
    * {
        animation: none !important;
        transition: none !important;
        scroll-behavior: auto !important;
    }
}

/* Contraste élevé */
@media (prefers-contrast: high) {
    :root {
        --text-color: #000000;
        --background-color: #ffffff;
        --link-color: #0000EE;
    }
}

États Interactifs


/* États des liens */
a {
    text-decoration: underline;
    text-underline-offset: 0.2em;
}

a:hover, a:focus {
    text-decoration-thickness: 2px;
}

/* États des boutons */
.button {
    padding: 0.5em 1em;
    min-height: 44px; /* Zone de tap minimale */
    min-width: 44px;

Bonnes pratiques :

  • Ne jamais supprimer complètement l'outline du focus
  • Utiliser des unités relatives pour la taille du texte
  • Tester avec des outils d'accessibilité
  • Vérifier la navigation au clavier

Container Queries

        gap: 1rem;
    }
}

    flex-direction: column;

Cascade Layers


/* Définition de l'ordre des layers */
@layer reset, base, theme, components, utilities;

/* Layer pour le reset */
@layer reset {
    * {
        margin: 0;
        padding: 0;
        box-sizing: border-box;
    }
}

/* Layer pour les styles de base */
@layer base {
    body {
        font-family: system-ui;
        line-height: 1.5;
    }
}

/* Layer pour les composants */
@layer components {
    .button {
        padding: 0.5em 1em;
    }
}
